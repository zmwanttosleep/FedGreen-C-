## 三、核心成果

### 3.1 月度电价与碳排放因子提取

从欧盟统计局月度数据 `monthly_full_release_long_format.csv` 中提取西班牙 2019‑2025 年：

- **月度电价**：`Day-ahead electricity price`（€/MWh） → 转换为 €/kWh。
- **月度碳排放强度**：`CO2 intensity`（gCO₂/kWh） → 转换为 kgCO₂/kWh。

生成文件：
- `decision/config/spanish_carbon_intensity.csv`（每年每月碳排值）
- `decision/config/eurostat_prices.csv`（每年每月电价值）

### 3.2 节点加权参数（月度版）

基于旧口径 `barcelona_ready_v1` 各节点 `sector_code` 频率，结合工业、商业、住宅、其他四类的电价/碳排基础值（工业 0.22€/0.24kg；商业 0.28€/0.22kg；住宅 0.18€/0.18kg；其他 0.23€/0.21kg），加权计算每个节点 2019‑2025 年每月加权电价和碳排放因子，生成：

- `decision/config/node_weighted_params_monthly.csv`（3444 行，41 节点 × 7 年 × 12 月）

**验证**：
```bash
head -5 decision/config/node_weighted_params_monthly.csv
```
输出显示各节点每月电价、碳排均无缺失。

### 3.3 动态阈值（按时段/节假日）

基于新口径测试集（`barcelona_ready_2023_2025`）所有节点的 `Valor`，按 `hour_code` 和 `is_holiday` 分组计算 10% 和 90% 分位数，生成：

- `decision/config/thresholds_dynamic.json`

**示例内容**：
```json
{
  "thresholds": [
    {"hour_code": 0, "is_holiday": 0, "low": 289.4, "high": 68432.5},
    ...
  ]
}
```

### 3.4 微调脚本优化（五星级）

重构微调脚本 `versions/v2_holiday_sector/train_federated_finetune_optimized.py`，新增：

- 早停机制（验证集真实 sMAPE 连续 10 轮不降则停止）
- 学习率调度（`ReduceLROnPlateau`，因子 0.5，耐心 5）
- 梯度裁剪（`max_norm=1.0`）
- 自动加载预训练模型
- 输出最佳模型至 `decision/models/model_fed_finetune.pth`
- 日志记录至 `results/two_stage/finetune_log.txt`

### 3.5 统一决策脚本（Phase 1 完成标志）

开发 `decision/scripts/run_decision.py`，具备以下功能：

| 参数 | 含义 | 示例 |
|------|------|------|
| `--window` | 窗口长度 | `1day`（4步输入）或 `7day`（28步输入） |
| `--mode` | 输出模式 | `all`（预测4个时段）、`first`（仅第一个时段） |
| `--model` | 自定义模型路径 | 可选，用于对比不同模型 |
| `--output_dir` | 输出目录 | 默认 `decision/outputs/` |
| `--no-plots` | 跳过图表生成 | 用于快速运行 |

**输出文件**：
- CSV：`decision_results_{window}_{mode}.csv`（详细决策记录）
- JSON：`summary_stats_{window}_{mode}.json`（总节省能耗、成本、碳排、准确率、延迟）
- 图表（自动生成）：
  - 累计节省能耗曲线
  - 各节点节省成本柱状图
  - 各节点碳减排柱状图
  - 决策准确率饼图
  - 决策类型分布图
  - 各时段平均节省能耗图（仅 `mode=all`）

**运行示例**：
```bash
# 七天窗口，全部时段决策，生成图表
python decision/scripts/run_decision.py --window 7day --mode all

# 一天窗口，仅第一个时段（未来6小时），不生成图表
python decision/scripts/run_decision.py --window 1day --mode first --no-plots
```

### 3.8 概念漂移检测规划

为增强模型自适应性，计划引入 ADWIN 算法，在预测误差序列上检测概念漂移。具体设计：

1. 使用微调后的模型，按时间顺序对新口径测试集进行预测，记录每个时段的绝对误差。
2. 对每个节点的误差序列运行 `river.drift.ADWIN()`，记录漂移点。
3. 可视化漂移点，并说明其对应真实事件（如季节变化、疫情等）。
4. 可选：在漂移点后，用最近数据重新微调模型，对比漂移前后精度。

此模块独立于决策引擎，可在一周内完成，作为复试第二创新点。

## 六、错误记录

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 月度电价文件 `monthly_full_release_long_format.csv` 中 `CO2 intensity` 字段无数据 | 数据更新至 2026‑02，但 2025‑2026 年数据缺失 | 使用现有数据插值，或忽略最近月份；对缺失月份用最近值填充 |
| `node_weighted_params_monthly.csv` 生成时 `price_euro_kwh` 为负数 | 欧盟统计局电价中包含负电价（电力市场特有） | 保留原值，决策时负电价会导致成本节省为负，符合实际 |
| `run_decision.py` 中 `generate_visualizations` 函数因 `matplotlib` 缺失导致崩溃 | 环境未安装绘图库 | 添加 `try/except` 捕获 `ImportError`，给出警告但不中断 |
| 一天窗口可学习时段脚本运行时 `hour_weights` 维度与输入不匹配 | `hour_code` 取值 0‑3，但权重参数为 (4,) 正确，检查索引时发现输入为 numpy 数组未转为 long | 确保 `hour_seq = torch.LongTensor(hour_seq)` |
