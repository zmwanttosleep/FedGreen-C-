## 三、核心原理（类比+专业）

### 3.1 v1 特征体系 vs v2.5 特征体系

**生动类比：**
- v1 = 只看基础信息（能耗+部门+节假日+周末）
- v2.5 = 加了很多细节（滞后7天、滚动统计、交互特征）

**实际效果：**
| 版本 | 特征数 | 参数量 | sMAPE | 结论 |
|:---|:---:|:---:|:---:|:---|
| v2.5 原版 | 19 | 460k | 70.34% | 过拟合 |
| v2.5 精选 | 16 | 118k | 67.21% | 仍过拟合 |
| v2.5 超级精选 | 12 | 54k | 68.17% | 仍无效 |
| **v1** | **7** | **451k** | **61.73%** | **最优** |

**专业解释：**
6小时粒度数据的信息量有限，滞后/滚动特征无法捕捉有效模式，反而引入噪声导致过拟合。v1 的基础特征（能耗+部门+节假日+周末）已经足够，更多特征只会降低泛化能力。

### 3.2 联邦学习配置原理

| 参数 | 值 | 专业依据 |
|:---|:---:|:---|
| `rounds=20` | 20轮 | 单节点20轮收敛，联邦需更多通信 |
| `local_epochs=5` | 5轮 | 平衡本地学习与全局聚合，防止过拟合 |
| `mu=0.001, 0.01` | 两个值 | 对比 FedProx 正则化效果 |
| 节点数=24 | 24个 | 取前24个节点（8001-8024），覆盖不同数据量 |

## 七、错误记录

| 错误 | 原因 | 解决方案 |
|:---|:---|:---|
| `TypeError: get_node_data_loader() got unexpected keyword argument 'holiday_feature'` | 训练脚本传递了 v1 不支持的参数 | 创建 `train_single_node_v1.py` 独立脚本 |
| `input.size(-1) must be equal to input_size. Expected 1, got 3` | 模型输入维度写死为 1 | 动态获取 `input_dim` |
| `JSONDecodeError: Expecting property name enclosed in double quotes` | JSON 格式错误（重复字段） | 用 Python 重新生成正确 JSON |
| `UnicodeEncodeError: 'gbk' codec can't encode character '\u2705'` | 特殊字符无法输出 | 移除 `✅⚠️` 等特殊字符 |
| `NameError: name 'min_epochs' is not defined` | 早停参数未传递 | 使用闭包捕获参数 |
