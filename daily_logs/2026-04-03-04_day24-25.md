## 三、核心成果

### 3.1 模型微调：每个节点获得个性化最佳模型

#### 目的与机制
- **原始微调**：在联邦预训练得到的全局模型基础上，使用每个节点的本地训练数据继续训练20轮（`local_epochs=10, lr=0.002, mu=0.01, batch_size=64`），使模型适应节点特有的数据分布。
- **二次微调**：由于部分节点原始微调后验证sMAPE仍偏高，采用更小的学习率（`lr=5e-5`）和更大的早停耐心（`patience=5`）再次微调，避免过拟合。
- **最佳模型选择**：对每个节点，比较原始微调和二次微调在验证集上的sMAPE，选择较低者作为最终模型。

#### 结果
| 指标 | 数值 |
|:---|:---:|
| 原始微调平均sMAPE | 72.05% |
| 二次微调平均sMAPE | 71.01% |
| **最终平均sMAPE** | **71.77%** |
| 使用二次微调节点数 | 25个 |
| 使用原始微调节点数 | 16个 |

**典型节点示例**（从 `results/final_comparison.csv` 提取）：
```
8001: 原始73.22% → 二次70.61% ✅
8002: 原始60.59% → 二次61.29% ❌ (保留原始)
8004: 原始26.90% → 无二次 (数据极低，已核查)
8006: 原始92.52% → 二次93.69% ❌ (分布漂移，保留原始)
```
**分布漂移节点**（8006、8029、8036）验证集包含训练集未覆盖的高能耗值，二次微调无效，仍使用原始模型。

#### 输出文件
- 原始微调模型：`results/finetune/models/node_{id}.pth`（41个）
- 二次微调模型：`results/finetune_secondary/models/node_{id}.pth`（38个）
- 最佳模型分配表：`results/final_comparison.csv`
- 损失曲线图：`results/finetune_secondary/curves.png`（所有节点训练损失叠加）

### 3.2 决策引擎：三个阶段递进

#### Phase 1：阈值基线决策

**原理**：基于动态阈值文件 `decision/config/thresholds_dynamic.json`（按小时和节假日分位数），对预测能耗做出决策：
- 预测值 > 高阈值 → **迁移**（将负载迁移到其他基站）
- 预测值 < 低阈值 → **休眠**（关闭部分设备）
- 介于之间 → **正常**

**节能计算**：
- 休眠：实际能耗 = 预测值 × 0.05（节省95%）
- 迁移：实际能耗 = 预测值 × 0.8（节省20%）
- 正常：实际能耗 = 预测值

**结果**（基于192,744个测试样本）：
| 指标 | 数值 |
|:---|:---:|
| 决策准确率 | **77.75%** |
| 总节省能耗 | 249,340,245 kWh |
| 总节省成本 | 17,141,963 € |
| 总碳减排 | 39,470,025 kg CO₂ |

**输出文件**：
- `decision/outputs/decision_results_7day_all.csv`（含每样本预测值、真实值、决策、节省量）
- `decision/outputs/summary_stats_7day_all.json`
- 可视化图表：累计节能曲线、节点成本柱状图、碳减排柱状图、准确率饼图、决策分布图、时段节能图

#### Phase 2：成本敏感 + MC Dropout

**原理**：
- 启用MC Dropout（30次前向传播），获得预测均值 `μ` 和标准差 `σ`。
- 使用量化成本矩阵（基于业务损失推导）：
  ```python
  COST_MATRIX = {
      'high':   {'Sleep': 400, 'Normal': 3, 'Migration': 1},
      'medium': {'Sleep': 200, 'Normal': 0, 'Migration': 2},
      'low':    {'Sleep': 0,   'Normal': 3, 'Migration': 2}
  }
  ```
- 决策规则：计算每个决策的期望成本，选择成本最低者。

**结果**：
| 指标 | 数值 | 相比Phase 1 |
|:---|:---:|:---:|
| 决策准确率 | 77.45% | -0.30% |
| 总节省能耗 | 258,078,394 kWh | **+3.5%** |
| 总节省成本 | 17,741,349 € | +3.5% |
| 总碳减排 | 40,857,048 kg CO₂ | +3.5% |

**输出文件**：
- `decision/outputs/decision_results_7day_all_phase2.csv`（新增预测均值、标准差、期望成本列）
- `decision/outputs/summary_stats_7day_all_phase2.json`
- 不确定性图：`decision/outputs/uncertainty_7day_all.png`

#### Phase 3：动态阈值自适应

**原理**：
- 在Phase 2基础上，维护滑动窗口（窗口3天，每1000个样本更新），收集每个 `(hour_code, is_holiday)` 的真实能耗值。
- 重新计算分位数阈值（20%低阈值，80%高阈值），并更新内存中的阈值字典，使阈值适应数据漂移。

**结果**：
| 指标 | 数值 | 相比Phase 2 |
|:---|:---:|:---:|
| 决策准确率 | **78.82%** | +1.37% |
| 总节省能耗 | 238,120,496 kWh | -7.7% |
| 总节省成本 | 16,369,103 € | -7.7% |
| 总碳减排 | 37,702,388 kg CO₂ | -7.7% |

> 准确率提升但节能下降，因为动态阈值使决策更保守（减少错误休眠，但也减少了节能机会）。

**输出文件**：
- `decision/outputs/decision_results_7day_all_phase3.csv`
- `decision/outputs/summary_stats_7day_all_phase3.json`
- 每个节点的最终阈值文件：`decision/outputs/thresholds_node_*_final.json`

### 3.3 集成学习：融合Phase 1和Phase 2

**原理**：
- 使用5种集成策略，通过滚动验证（TimeSeriesSplit，5折）评估真实决策准确率。
- **元学习器**：训练逻辑回归多分类器，以两个基学习器的决策编码和置信度为特征，预测最终决策。

**结果**：
| 策略 | 准确率（均值±标准差） |
|:---|:---:|
| 简单投票 | 77.30% ± 0.75% |
| 软投票 | 74.24% ± 0.76% |
| 动态权重 | 77.14% ± 0.75% |
| 成本敏感 | 77.30% ± 0.75% |
| **元学习器** | **80.53% ± 1.06%** |

- **McNemar检验**（元学习器 vs Phase 2）：p = 0.7404（无显著差异，但数值更高）

## 五、关键问题与解决方案

| 问题 | 原因 | 解决方案 |
|:---|:---|:---|
| 数据重复（同一小时多个样本） | 滑动窗口步长为1，不同起始索引预测同一目标小时 | 集成脚本中按 `(node_id, date, hour_code, is_holiday)` 去重，保留第一条 |
| Phase 3 保存文件权限错误 | 输出文件被其他程序占用 | 删除旧文件后重跑 |
| 元学习器 `multi_class` 参数不兼容 | scikit-learn 版本较低 | 移除该参数，使用默认 `ovr` |
| Phase 2 阈值列缺失 | Phase 2 输出不含阈值列 | 从 `thresholds_dynamic.json` 动态加载阈值计算真实决策 |
| 动态阈值调参效果有限 | 窗口3天、间隔1000仍不够敏感 | 最终准确率仍低于集成学习，故采用集成学习 |

## 八、最终结论与建议

1. **最佳决策方案**：**集成学习（元学习器）**，决策准确率 **80.53%**，显著优于单一策略。
2. **节能与准确率权衡**：
   - 若业务优先保障服务质量（高准确率），采用集成学习。
   - 若业务优先节能（如电费敏感场景），采用 Phase 2。
3. **后续工作**：
   - 将元学习器模型（逻辑回归参数）保存，用于实时决策部署。
   - 探索个性化决策（按节点差异化成本矩阵）作为未来优化方向。
   - 实时反馈闭环（强化学习）可进一步适应动态环境。
