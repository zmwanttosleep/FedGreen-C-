## 三、核心成果

### 3.1 SHAP 窗口分析（单节点 → 批量验证）

#### 3.1.1 单节点修复与初步发现

**目标**：解释短期窗口（1天）为何优于长期窗口（7天）。  
**方法**：对节点8001分别训练1天窗口（4步）和7天窗口（28步）的 LSTM 模型，使用 `shap.KernelExplainer` 计算各时间步的特征重要性。

**关键错误与修复**：
- **错误**：原始脚本 `shap_window_comparison_optimized.py` 在计算 7 天窗口 SHAP 值时，`shap_mean` 形状为 `(4,)` 而非 `(28,)`，导致 `plt.bar` 绘图时 `shape mismatch`。
- **原因**：`shap_values` 是列表，长度为输出维度（4），每个元素形状 `(n_samples, window_size)`。原代码 `np.mean(shap_values, axis=0)` 错误地将列表视为数组，导致聚合后形状错误。
- **修复**：将 `shap_values_array = np.array(shap_values)`，再 `np.mean(np.abs(shap_values_array), axis=(0,1))`，直接得到 `(window_size,)` 的重要性数组。
- **验证**：修复后正确生成 7 天窗口各时间步重要性（峰值集中在第一天）。

**生成文件**（保存在 `results/figures/`）：
- `shap_1day_timesteps.png` — 1天窗口各时间步重要性（峰值在12-18点）
- `shap_1day_waterfall.png` — 单个样本的 SHAP 力图
- `shap_window_report.txt` — 分析报告（含关键观察）

#### 3.1.2 多节点批量 SHAP 分析（核心突破）

**目的**：验证“短期窗口优于长期窗口”的结论在多个节点上是否具有普遍性，并自动生成汇总图表。

**设计**：
- **抽样依据**：读取聚类文件 `results/barcelona_clustering/node_classification.csv`，按 `cluster` 类别分层抽样，每类 3 个，共选中 23 个节点（排除数据异常节点）。
- **训练与 SHAP**：对每个节点分别训练 1天窗口（4步）和 7天窗口（28步）模型（10 轮，固定 μ=0.05），计算 SHAP 时间步重要性。
- **汇总**：收集各节点 sMAPE 和 SHAP 数组，生成箱线图、热力图、CSV 表格。

**脚本**：`versions/v2_holiday_sector/batch_shap_by_cluster.py`（15419 字节，支持自动路径、异常处理、日志）

**运行命令**：
```bash
cd /d/Desk/desk/beiyou_c_project/versions/v2_holiday_sector
python batch_shap_by_cluster.py --samples_per_cluster 3
```

**输出目录**：`versions/v2_holiday_sector/results/shap_multi_nodes/`

**生成成果**：
- 每个节点的 SHAP 数组（`node_XXXX_shap_1d.npy`，`node_XXXX_shap_7d.npy`），共 23 个节点。
- 汇总图表：
  - `smape_comparison_boxplot.png` — 1天 vs 7天窗口 sMAPE 箱线图
  - `daily_importance_heatmap.png` — 各节点 7 天窗口逐日重要性热力图
- `summary.csv` — 各节点 sMAPE 与验证损失
- 运行日志（`batch_shap_20260327_204844.log`，46MB）

**关键统计数据**（基于 `summary.csv`）：
| 指标 | 1天窗口 | 7天窗口 | 差异 |
|------|---------|---------|------|
| sMAPE 均值 | 38.5% | 41.2% | **+2.7%** |
| sMAPE 中位数 | 36.8% | 40.1% | **+3.3%** |
| 优于短期节点占比 | — | — | **78% (18/23)** |

**可视化结论**：
- **箱线图**：1天窗口的整体分布明显低于 7天窗口，且离群点更少，稳定性更高。
- **热力图**：7天窗口中，**第一天的重要性平均是后续天数的 2.3 倍**，且随天数递增快速衰减，验证了“近期数据主导预测”的普遍规律。

**问题记录**：

- 部分节点 7 天窗口训练损失未完全收敛（10 轮可能不足），后续可增加轮数。

### 3.2 双流模型（7天窗口 + 清华数据）

**目的**：尝试利用清华细粒度数据提升巴塞罗那7天窗口预测精度，验证短期窗口优势是否可被外部数据弥补。

**设计**：双流 LSTM，分别处理巴塞罗那28步输入和清华4步输入，融合后输出。  
**脚本**：`train_federated_dual_stream_7day.py`（15317 字节）及加权版本。

**2节点验证结果**（记录于 Day 17 日志）：
- 7天不加权基线：55.42%
- 双流融合：51.90%（提升 3.52%）
- 双流+时段加权：无效（损失震荡）

**结论**：清华数据可小幅改善 7 天窗口精度，但仍远不及 1 天窗口（28.45%），证明短期窗口的优越性无法通过外部数据弥补。

## 六、错误记录

| 错误 | 原因 | 解决方案 |
|:---|:---|:---|
| `shape mismatch: objects cannot be broadcast to a single shape` | SHAP 值聚合时未正确将输出维度与时间步维度分离 | 改用 `np.mean(np.abs(shap_values_array), axis=(0,1))` 直接聚合 |
| 部分节点 SHAP 计算失败（如8022） | 数据异常导致模型训练崩溃 | 脚本添加 try-except，跳过并记录日志 |
| 双流模型损失震荡 | 时段加权实现错误 | 放弃加权，保留纯双流 |
| 批量分析中 sMAPE 出现 0% | 某些节点验证集预测值为 0 导致除零 | 添加 epsilon=1e-8 保护 |
